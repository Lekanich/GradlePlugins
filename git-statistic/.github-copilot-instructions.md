# Git Statistic Plugin - Development Guide

## Overview

Git Statistic Plugin generates comprehensive Git statistics, status reports, and interactive dashboards. This is a **separate plugin** from git-tool, focused on **reporting and visualization** rather than Git operations.

## Key Differences from git-tool

| Feature | git-tool | git-statistic |
|---------|----------|---------------|
| **Purpose** | Execute Git operations | Generate reports & dashboards |
| **Tasks** | Action-based (create, push, delete) | Report-based (generate, analyze) |
| **Output** | Git state changes | HTML/JSON/Markdown reports |
| **Use Case** | Automation & CI/CD operations | Visualization & analysis |

## Architecture

### Core Components

1. **GitStatisticPlugin**: Main plugin class, registers tasks
2. **GitStatisticExtension**: Configuration DSL
3. **GitStatusReportTask**: Generate status reports (JSON/HTML/Markdown)
4. **GitDashboardTask**: Generate interactive HTML dashboard
5. **Data Models**: GitStatusData, RepositoryInfo, CommitInfo, etc.

### Task Hierarchy

```
DefaultTask
├── GitStatusReportTask (reports)
└── GitDashboardTask (visualization)
```

Unlike git-tool which extends `Exec`, these tasks extend `DefaultTask` because they:
- Execute multiple Git commands internally
- Process and format data
- Generate output files (not just run Git commands)

## Code Conventions

### Task Implementation

```kotlin
abstract class MyReportTask : DefaultTask() {
    @get:Input
    abstract val configProperty: Property<String>
    
    @get:OutputFile // or @OutputDirectory for dashboards
    abstract val outputFile: RegularFileProperty
    
    @TaskAction
    fun generateReport() {
        val data = collectGitData()
        writeReport(data)
    }
    
    private fun executeGit(vararg args: String): String {
        val output = ByteArrayOutputStream()
        project.exec { spec ->
            spec.commandLine("git", *args)
            spec.standardOutput = output
            spec.isIgnoreExitValue = true
        }
        return output.toString(Charsets.UTF_8).trim()
    }
}
```

### Data Collection Pattern

Always use `executeGit()` helper method for consistency:

```kotlin
private fun executeGit(vararg args: String): String {
    val output = ByteArrayOutputStream()
    val result = project.exec { spec ->
        spec.commandLine("git", *args)
        spec.standardOutput = output
        spec.errorOutput = ByteArrayOutputStream() // Suppress errors
        spec.isIgnoreExitValue = true
    }
    
    if (result.exitValue != 0) {
        throw GradleException("Git command failed: git ${args.joinToString(" ")}")
    }
    
    return output.toString(Charsets.UTF_8).trim()
}

private fun executeGitOrNull(vararg args: String): String? {
    return try {
        executeGit(*args).takeIf { it.isNotBlank() }
    } catch (e: Exception) {
        null
    }
}
```

### Testing Approach

Since tasks generate reports from Git data:

1. **Unit Tests**: Test plugin registration and configuration
2. **Integration Tests**: Test with real/mock Git repositories
3. **Output Validation**: Verify report content and format

Example test:

```kotlin
@Test
fun `task generates JSON report`() {
    val project = ProjectBuilder.builder().build()
    project.plugins.apply("io.github.lekanich.git-statistic")
    
    val task = project.tasks.getByName("gitStatusReport") as GitStatusReportTask
    task.format.set(ReportFormat.JSON)
    task.outputFile.set(project.file("build/status.json"))
    
    // Execute task (requires Git repository)
    // Verify output file exists and contains valid JSON
}
```

## Report Formats

### JSON Format

**Purpose**: Machine-readable for CI/CD integration

**Structure**:
```json
{
  "reportGeneratedAt": "ISO-8601 timestamp",
  "repository": { "currentBranch", "upstreamBranch", "branchStatus" },
  "commit": { "hash", "message", "author", "date", "totalCount" },
  "workingDirectory": { "isClean", "modified", "untracked", "staged", "conflicts" },
  "tags": { "latest", "commitsSinceLatest", "all" },
  "remote": { "name", "url", "reachable" }
}
```

### HTML Format

**Purpose**: Human-readable with styling

**Features**:
- Styled tables
- Color-coded status badges
- Expandable file lists
- Responsive design

### Markdown Format

**Purpose**: Documentation-friendly

**Use Case**: README injection, release notes, documentation

## Configuration Cache Compatibility

✅ **Tasks are configuration cache compatible:**
- No `Project` captured in task actions
- All inputs properly annotated with `@Input`
- All outputs properly annotated with `@OutputFile` or `@OutputDirectory`
- Use `Provider` and `Property` types

## Performance Considerations

### Expensive Operations

1. **Commit History**: Limited by `commitHistoryDepth` (default: 50)
2. **File Changes**: Can be slow on large repositories
3. **Remote Checks**: Network-dependent

### Optimization Strategies

```kotlin
gitStatistic {
    // Disable expensive operations if not needed
    includeRemoteStatus.set(false)      // Skip network calls
    includeFileHeatmap.set(false)       // Skip file analysis
    commitHistoryDepth.set(20)          // Reduce history depth
}
```

## Common Patterns

### Multi-Format Reports

Generate multiple formats in one build:

```kotlin
tasks.register<GitStatusReportTask>("statusJson") {
    format.set(ReportFormat.JSON)
    outputFile.set(layout.buildDirectory.file("reports/status.json"))
}

tasks.register<GitStatusReportTask>("statusHtml") {
    format.set(ReportFormat.HTML)
    outputFile.set(layout.buildDirectory.file("reports/status.html"))
}
```

### CI/CD Integration

```kotlin
tasks.register("ciReport") {
    dependsOn("gitStatusReport")
    doLast {
        val statusFile = layout.buildDirectory.file("reports/git-status/status.json").get().asFile
        // Parse and use in CI logic
    }
}
```

### Build Artifact Embedding

```kotlin
tasks.named<Jar>("jar") {
    dependsOn("gitStatusReport")
    from(layout.buildDirectory.file("reports/git-status/status.json")) {
        into("META-INF/git")
    }
}
```

## Troubleshooting

### Common Issues

**"Git command failed"**
- Ensure Git is installed and in PATH
- Verify you're in a Git repository
- Check Git command syntax

**"Remote not reachable"**
- Network connectivity issue
- Set `includeRemoteStatus.set(false)` to skip

**"Task output not generated"**
- Check task execution logs
- Verify output directory permissions
- Look for Git errors in build output

## Best Practices

1. **Report Generation**: Run before builds for artifact embedding
2. **Dashboard**: Run manually or on-demand (expensive)
3. **CI/CD**: Use JSON format for parsing
4. **Documentation**: Use Markdown format
5. **Human Review**: Use HTML format or dashboard
